go之第一篇

一、文件名、关键字、标识符

​ go的文件一般以.go结尾，一般由小写字母构成，如果文件名由多个部分组成，则使用下划线_对他们进行分组。文件名不含空格和其他特殊字符。

​ go的文件大小并没有限制，一个源文件可以由很多行代码构成。

​ 在go语言内部区分大小写。

​ 标识符由_、数字，字母构成，不能以数字开头。

​ _是一个特殊的标识符，我们称之为空白标识符，他可以像其他标识符一样用于声明和赋值，但赋给它的值都被抛弃，不能在后续的代码中使用，也不能给其他标识符赋值或者参与运算。

​ 一共有25个关键字：

break default func interface select
case defer go map struct
chan else goto package switch
const fallthrough if range type
continue for import return var

​ 程序一般由关键字、常量、变量、运算符、类型和函数组成。

​ 程序中可能会使用到这些分隔符：括号 ()，中括号 [] 和大括号 {}。

​ 程序中可能会使用到这些标点符号：.、,、;、: 和 …。

​ 程序的代码通过语句来实现结构化。每个语句不需要像 C 家族中的其它语言一样以分号 ; 结尾，因为这些工作都将由 Go 编译器自动完成。

​ 如果你打算将多个语句写在同一行，它们则必须使用 ; 人为区分，但在实际开发中我们并不鼓励这种做法。

二、基本结构和组成

​ 包可以由好几个源程序构成，在每个源程序第一行必须说明它属于哪个包。

​ 每个go应用程序可以由好几个包构成，并且必须含有一个叫做mian的包。

​ 如果对一个包进行更改或重新编译，所有引用了这个包的客户端程序都必须全部重新编译。

​ Go 中的包模型采用了显式依赖关系的机制来达到快速编译的目的，编译器会从后缀名为 .o 的对象文件（需要且只需要这个文件）中提取传递依赖类型的信息。

  1. 包名

  2. 导入其他包

  3. 定义一些变量

  4. 定义一些函数

    左大括号 { 必须与方法的声明放在同一行，这是编译器的强制规定。

    右大括号 } 需要被放在紧接着函数体的下一行。如果你的函数非常简短，你也可以将它们放在同一行；

    main 函数是每一个可执行程序所必须包含的，一般来说都是在启动后第一个执行的函数（如果有 init() 函数则会先执行该函数）。

    main 函数既没有参数，也没有返回类型。

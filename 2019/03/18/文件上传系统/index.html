文件上传系统

发表于 2019-03-18 | 分类于 项目介绍

1，文件上传系统的一些基本实现的功能如下

  1. 在服务器务器的文件下将看到客户端上传的文件或者上传的文件夹，并且文件的目录结构保持；
  2. 在客户端可以上传文件或者文件夹，并且保留文件夹结构；
  3. 文件传输过程种使用ASE加密算法进行了加密；
  4. 使用多线程编程，多个客户端可以同时向服务器传输文件；

3，各文件详细介绍

​ 1，ase1.java文件：

​ 这个文件中主要实现了加解密函数，以及将原始数据转化成byte的函数，由于当时并不懂计算机安全的一些知识，更不懂ASE算法的相关原理，所以这个文件中的加解密代码基本上来自于网络，所以暂时不做多的解释，等以后自己会重写这部分代码。

​ 2，myclient.java文件中：

​ 这个文件主要是上传客户端的入口，通过命令行启动本程序，这个程序会读取命令行的参数，确定上传的文件，之后new一个新的线程出来，作为一个上传客户端，并且会创建一个指定ip和端口的socket。

​ 需要注意的是这个加密算法将1024个字节加密成1040个，这个很坑

​ 3，clinetThread.java文件：

​ 这个类主要implements 了Runnable类，使用的是实现接口的方法。

​ 1).定义一个类实现Runnable接口，重写接口中的run()方法。在run()方法中加入具体的任务代码或处理逻辑。

(2).创建Runnable接口实现类的对象。

​ (3).创建一个Thread类的对象，需要封装前面Runnable接口实现类的对象。（接口可以实现多继承）

​ (4).调用Thread对象的start()方法，启动线程

​ 在实现本类的run方法时：

​ 1，首先我们判断需要传输的是文件还是文件夹，如果是传输文件夹，这时我们采取将文件夹压缩成zip文件的方法，统一传输的文件格式；并且将这是文件夹的信息压入字节流；

​ 2，截取文件名，并且测量长度，将文件名长度和文件名同时放入BufferedOutputStream流中；

​ 3，测量文件长度，将它放到流里面；

​ 4，调用加密算法，对文件进行加密；

​ 5，将加密后的文件内容和文件长度放入流里面，对文件进行传输；

​ 4，myserver.java

​ 这个类提供了服务器接口，同样使用了多进程的编程方法。在这个类中，我们维护一个socketList来维护这个端口的所有socket，并且创建了一个ServerSocket来监听该接口是否有信息传输过来。之后启动一个进程去将文件存储在服务器端。

​ 5，MyServerThread.java

​ 在该类的run方法中：

​ 1，判断是文件还是文件夹

​ 2，读取文件名长度，读取文件名

​ 3，读文件长度

​ 4，读加密后的文件长度，计算需要解密的次数

​ 5，读取文件内容

​ 6，如果是文件夹，我们应该将文件夹解压，之后删除.zip文件

​ 6，fileset.java

​ 这个类主要是将文件夹当成一个区别于文件的个体去传输，即不是使用文件压缩的思路去做，这个方法将文件夹的目录结构也进行传输，最后调用再调用文件进程进行传输，但由于时间关系，并没有将服务器端的这种方法写出来，所以也仅仅作为一种思路放在这个博客里面。

​ 7，zip.java

​ 这个类主要实现文件的压缩和解压缩，代码参考了网络中的很多知识，所以也不做过多的解释。

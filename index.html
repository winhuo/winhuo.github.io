Go语言数组和切片

go语言的容器收到了python的很大影响

一、数组

​ 数组一般是各种编程语言的基本主力，但是go语言在数组的基础上创造了切片的概念，切片在go语言中随处可见。它们构建在数组之上并且提供更强大的能力和便捷。

数组中的元素可以通过下标获取，或者可以通过下标修改。但索引超过最大长度时，会报错。

​ Go 语言中的数组是一种 值类型（不像 C/C++ 中是指向首元素的指针），所以可以通过 new() 来创建： var arr1 = new([5]int)。数组的另一种声明方式是var arr2 [5]int，这两种是有区别的。arr1的类型是*[5]int,

但是arr2的类型是[5]int。

​ 所以赋值时，arr2 := *arr1。

​ 所以在函数中数组作为参数传入时，如 func1(arr2)，会产生一次数组拷贝，func1 方法不会修改原始的数组 arr2。

如果你想修改原数组，那么 arr2 必须通过&操作符以引用方式传过来，例如 func1(&arr2），下面是一个例子

​ *数组常量的初始化数组：

​ var arrLazy = [...]int{5, 6, 7, 8, 22}

​ var arrAge = [5]int{18, 20, 15, 22, 16}

​ var arrKeyValue = [5]string{3: "Chris", 4: "Ron"}

​ *将数组传递给函数可以有两种方法，第一种是传递数组的指针，第二种是使用数组的切片。

二、切片

​ 切片（slice）是对数组一个连续片段的引用（该数组我们称之为相关数组，通常是匿名的），所以切片是一个引用类型。和数组不同的是，切片的长度可以在运行时修改，最小为 0 最大为相关数组的长度：切片是一个 长度可变的数组。

切片提供了计算容量的函数 cap() 可以测量切片最长可以达到多少：它等于切片的长度 + 数组除切片之外的长度。如果 s 是一个切片，cap(s) 就是从 s[0] 到数组末尾的数组长度。切片的长度永远不会超过它的容量，所以对于 切片 s 来说该不等式永远成立：0 <= len(s) <= cap(s)。

声明切片的格式是： var identifier []type（不需要说明长度）

切片的初始化格式是：var slice1 []type = arr1[start:end]

切片在内存中的组织方式实际上是一个有 3 个域的结构体：指向相关数组的指针，切片长度以及切片容量。

在数组还没有声明之前，如果我们想要使用切片，我们可以使用make（）函数创建一个切片。

所以定义 s2 := make([]int, 10)，那么 cap(s2) == len(s2) == 10。

make 接受 2 个参数：元素的类型以及切片的元素个数。

  • new(T) 为每个新的类型T分配一片内存，初始化为 0 并且返回类型为*T的内存地址：这种方法 返回一个指向类型为 T，值为 0 的地址的指针，它适用于值类型如数组和结构体；它相当于 &T{}。
  • make(T) 返回一个类型为 T 的初始值，它只适用于3种内建的引用类型：切片、map 和 channel。

换言之，new 函数分配内存，make 函数初始化；

函数

每一个程序包含很多函数，函数是基本的代码块。

Go语言中有三种函数：普通的带有名字的函数，匿名函数或者lanbda函数，方法。

除了main函数和init函数，其他函数都能有参数和返回值。函数的参数，返回值以及类型被叫做函数的签名。

一、函数参数和返回值

​ 函数能够接受参数供自己使用，也能返回0个或者多个值，相对于其他语言，多返回值是Go语言的一个特性。

​ 函数定义时，它的形参一般是有名字的，不过我们也可以定义没有形参名的函数，只有相应的形参类型，就像这样：func f(int, int, float64)。

​ 没有参数的函数通常被称为 niladic 函数（niladic function），就像 main.main()

​ 值传递和引用传递的区别，值传递对形参的改变并不能改变原变量的值，如果希望可以改变原来的值，那么应该使用一个指针作为参数进行传递。几乎在任何情况下，传递指针的消耗都要被传递值得消耗小很多。

​ 但是如果一个函数需要返回很多个值，那么一般需要返回一个切片或者返回一个结构体。

​ 传递指针不但可以节约内存空间，还可以赋予函数直接修改外部变量得能力，所以被修改的变量并不需要返回。

二、传递变长的参数

匿名函数同样被称之为闭包（函数式语言的术语）：它们被允许调用定义在其它环境下的变量。闭包可使得某个函数捕捉到一些外部状态，例如：函数被创建时的状态。另一种表示方式为：一个闭包继承了函数所声明时的作用域。这种状态（作用域内的变量）都被共享到闭包的环境中，因此这些变量可以在闭包中被操作，直到被销毁。闭包经常被用作包装函数：它们会预先定义好 1 个或多个参数以用于包装，详见下一节中的示例。另一个不错的应用就是使用闭包来完成更加简洁的错误检查。

控制结构

一、if-else

即使当代码块之间只有一条语句时，大括号也不可被省略

关键字 if 和 else 之后的左大括号 { 必须和关键字在同一行，如果你使用了 else-if 结构，则前段代码块的右大括号 } 必须和 else-if 关键字在同一行。这两条规则都是被编译器强制规定的。

二、switch结构

变量 var1 可以是任何类型，而 val1 和 val2 则可以是同类型的任意值。类型不被局限于常量或整数，但必须是相同的类型；或者最终结果为相同类型的表达式。前花括号 { 必须和 switch 关键字在同一行。

如果在执行完每个分支的代码后，还希望继续执行后续分支的代码，可以使用 fallthrough 关键字来达到目的。

三、for结构

​ 1、如果想要重复执行某些语句，Go 语言中您只有 for 结构可以使用。不要小看它，这个 for 结构比其它语言中的更为灵活。

同样的，左花括号 { 必须和 for 语句在同一行，计数器的生命周期在遇到右花括号 } 时便终止。一般习惯使用 i、j、z 或 ix 等较短的名称命名计数器。

​ 2、for 结构的第二种形式是没有头部的条件判断迭代（类似其它语言中的 while 循环），基本形式为：for 条件语句 {}。

​ 3、for-range Go 特有的一种的迭代结构，您会发现它在许多情况下都非常有用。它可以迭代任何一个集合（包括数组和 map，详见第 7 和 8 章）。语法上很类似其它语言中 foreach 语句，但您依旧可以获得每次迭代所对应的索引。一般形式为：for ix, val := range coll { }。

​ *val 始终为集合中对应索引的值拷贝，因此它一般只具有只读性质，对它所做的任何修改都不会影响到集合中原有的值（译者注：如果 val 为指针，则会产生指针的拷贝，依旧可以修改集合中的原值）

四、Break与continue

​ 一个 break 的作用范围为该语句出现后的最内部的结构，它可以被用于任何形式的 for 循环（计数器、条件判断等）。但在 switch 或 select 语句中（详见第 13 章），break 语句的作用结果是跳过整个代码块，执行后续的代码。

​ 关键字 continue 忽略剩余的循环体而直接进入下一次循环的过程，但不是无条件执行下一次循环，执行之前依旧需要满足循环的判断条件。

指针

​ Go 语言为程序员提供了控制数据结构的指针的能力；但是，你不能进行指针运算。通过给予程序员基本内存布局，Go 语言允许你控制特定集合的数据结构、分配的数量以及内存访问模式，这些对构建运行良好的系统是非常重要的：指针对于性能的影响是不言而喻的，而如果你想要做的是系统编程、操作系统或者网络应用，指针更是不可或缺的一部分。

​ Go 语言的取地址符是 &，放到一个变量前使用就会返回相应变量的内存地址。

然后使用 intP = &i1 是合法的，此时 intP 指向 i1。

你可以在指针类型前面加上 号（前缀）来获取指针所指向的内容，这里的 号是一个类型更改器。使用一个指针引用一个值被称为间接引用。

当一个指针被定义后没有分配到任何变量时，它的值为 nil。

在书写表达式类似 var p *type 时，切记在 * 号和指针名称间留有一个空格。

所以说，Go 语言和 C、C++ 以及 D 语言这些低级（系统）语言一样，都有指针的概念。但是对于经常导致 C 语言内存泄漏继而程序崩溃的指针运算（所谓的指针算法，如：pointer+2，移动指针指向字符串的字节数或数组的某个位置）是不被允许的。Go 语言中的指针保证了内存安全，更像是 Java、C# 和 VB.NET 中的引用。

go语言之前言

发表于 2019-06-12 | 分类于 go语言

一，go语言的历史和一些特性

1，go语言的历史

​ 随着软件行业的发展，对编程语言有了以下几个要求：

​ 1，能够快速的开发出软件

​ 2，开发出的软件能在多核计算机上良好运行

​ 3，开发出的软件能在网络环境下运行

​ 4，使人们享受开发的过程

​ go语言在这样的条件下诞生了，他继承了c，java，c#的一些优点，提倡用接口的方法处理面向对象，通过goroutine和channel来支持并发和并行编程。

​ go语言起源于2007年，在2009年对外发布。2009年将go语言对外开源。

  • 2007 年 9 月 21 日：雏形设计
  • 2009 年 11 月 10日：首次公开发布
  • 2010 年 1 月 8 日：当选 2009 年年度语言
  • 2010 年 5 月：谷歌投入使用
  • 2011 年 5 月 5 日：Google App Engine 支持 Go 语言

2，go语言的特性

​ go语言从本质上实现了并发编程

​ go没有类和对象的概念，通过接口的概念来实现多态

​ 函数是go语言中最基本的构建，使用灵活

​ go语言使用静态类型，类型安全

​ 隐式的类型转换不被允许

​ go语言支持交叉编译，可以在运行linux的电脑上开发运行在windows上的软件，完全支持UTF-8

3，go语言的用途

​ 应用于web服务器吗，存储集群，或者类似用途的巨型中央服务器，擅长高性能分布式系统。提供海量并行，擅长游戏服务器端的开发

​ 擅长所谓的复杂事务处理（CEP）

​ 垃圾回收和内存分配，适合开发实时性要求很高的软件

4，特性的缺失

  • 为了简化设计，不支持函数重载和操作符重载
  • 为了避免在 C/C++ 开发中的一些 Bug 和混乱，不支持隐式转换
  • Go 语言通过另一种途径实现面向对象设计（第 10-11 章）来放弃类和类型的继承
  • 尽管在接口的使用方面（第 11 章）可以实现类似变体类型的功能，但本身不支持变体类型
  • 不支持动态加载代码
  • 不支持动态链接库
  • 不支持泛型
  • 通过 recover 和 panic 来替代异常机制（第 13.2-3 节）
  • 不支持静态变量

集群和高可用

一、集群的概念和介绍

​ 大多数网络服务采用7*24的服务策略，所以我们将大量的主机有机的结合在一起，以满足当前大数据时代海量访问负载。

​ LVS(Linux Virtual Server)，linux虚拟服务器。在linux内核实现了基于IP的数据请求负载均衡调度方案，终端的web请求会发送给LVS调度器，调度器根据自己的算法确定将请求分配给后台的一台服务器。LVS的服务模式主要有NAT，TUN，DR三种模式。

​ NAT：LVS调度器使用两块网卡配置不同的IP地址，一块设置为私钥IP和内部设备互联，另一个设置为外网IP与外部网络互联。用户只能访问到外网的ip，称为VIP。这时候LVS会根据调度算法选择后端的一个真实服务器，真实服务器将响应数据包返回给调度器，调度器将信息返回用户。(维护了一个HASH表)

​ TUN:在NAT模式下，调度器是整个集群环境的瓶颈，我们还知道数据请求包远远小于数据响应包。所以TUN的思路就是将请求和响应分开，调度器负责处理请求，在调度器和真实服务器之间设置IP隧道（一种添加头部及的技术），服务器将响应包直接返回给用户。所以在这个模式下，集群环境中的真实服务器要求能和互联网进行互联。

​ DR：在TUN模式中，建立IP隧道依然会消耗一些服务器资源，所以推出了DR模式，LVS的作用依旧是根据算法选择真实服务器处理请求，要求调度器和后端服务器必须在同一个局域网，在真实服务器返回响应时，需要将源IP设置成VIP。

​ LVS的常见调度算法：

  • 轮询调度

  • 加权轮询调度

  • 最小连接调度

  • 加权最小连接调度

  • 局部性最少连接

  • 目标地址散列调度

  • 源地址散列调度

    LVS现在已经集成到了LINUX内核，整个LVS分为内核层和用户层，内核层负责核心算法的实现，用户层需要安装ipvsade。其内核模块名称为ip_vs。

二、Keepalived双机热备

​ keepalived是使用C语言编写的路由热备软件，主要为linux系统提供简单高效的负载均衡及高可用解决方案。我们可以使用keepalived来部署多个LVS调度器，仅有一台作为主服务器，其他调度器当作备用，当主设备发生故障时，keepalived可以自动将备用升级为主设备。

​ VRRP协议介绍：在静态路由环境下防止单点故障而设计的主从灾备协议，它将两台设备虚拟成一个设备，对外提供一个设备的IP地址，同一时刻仅有一个设备可以拥有该虚拟IP地址。主设备会一直向从设备发送信息，从设备收不到时，立即升级自己的角色。

​ VRRP协议有个特殊情况，当虚拟IP和其中一个设备的ip相同时，它将永远是主设备状态。

​ keepalived可以使用yum安装，之后按照需求设置他的配置文件。

三、squid代理服务器

​ Squid是一个支持http，https，ftp等服务的web缓存代理软件，他可以通过缓存来降低带宽占用并优化页面响应时间。此外他还具有强大的访问控制功能。

​ 工作流程：客户端访问squid代理服务器，之后它去向真实服务器请求，并且响应客户端，之后将这部分内容保存下来。

​ 常见的代理服务类型：

​ 正向代理服务器：内部网页希望获取外部网络时缓存的页面数据。所有的刻画u电脑通过设置代理服务器连接至Squid，通过Squid代理上网，它主要提供缓存加速和访问控制。

​ 透明代理，透明代理和正向代理类似，区别时正向代理需要对每个客户端进行代理设置透明设置不需要终端用户进行设置，它结合网关进行设置，对用户透明。

​ 反向代理，反向代理结合智能DNS实现基本的CDN(内容分发网络)，通过DNS的试图功能，我们可以为不同地区的DNS请求返回不同的结果给客户端。

四、HAProxy负载均衡

​ HAProxy的工作模式使其可以非常简单和安全的集成到我们现有的站点架构中，可以对外屏蔽内部真实的WEB服务器，防止服务器遭受到攻击。

五、Nginx高级应用

​ 它除了可以作为后端的http服务器外，他还是一个高性能的反向代理服务器，他可以根据规则以轮询，ip哈希，url哈希等方式调度后台的真实服务器，Nginx也支持对后端服务器的健康检查功能。

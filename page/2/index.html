记腾讯实习生

发表于 2019-04-09 | 分类于 生活

记2019腾讯提前批后台开发offer历程（CSIG）

1，背景

​ 三月初，经过和一些读研学长以及工作学长的交流，加上自己对考研数据的考量，最重要的询问内心，看清自己究竟想要追求什么，决定结束自己的校园生活，开始找工作，进入社会。在社会这个更大的平台上继续努力，挑战人生中的另一段旅途。

2，简历投递

​ 3月7日第一次简历进行投递，在腾讯的招聘官网进行信息补充，简历是被一个学长进行内推的，事业群不明确。简历中写了大学的必修课成绩，大学做过的几个大小项目，大学在其他方面取得的成绩，以及获得的奖励。

3，准备面试

​ 编程方面，开始刷力扣，每天一到两道，主要是熟悉一下语言。之后学习了自己之前没搞清楚的动态规划算法，之后系统学习了STL的那些容器和迭代器，掌握了常用的STL函数。

​ 数据结构和算法，复习了所有的查找和检索算法（很详细），之后复习图的一些算法（最小生成树，最短路径），各种树（主要是红黑树，b，b+，avl等），树的遍历算法等，之后复习了队列，链表，栈等。基本算是把主要内容过了一遍，还了解了各种树的应用。

​ 操作系统主要复习了进程管理和内存管理。了解了linux的调度算法，基本除了文件管理也过了一遍。

​ 计算机网络，除了物理层，每一层都搞得很明白，应用层也很重要。

​ 数据库，常用的语句，事务的特性，事务的锁，数据库的底层存储。

​ c++，复习了那些难点，比如虚函数，静态变量，存储空间等各个方面，基本把语言过了一遍。

​ 网络安全和密码学，主要是公私钥加密还有CA部分。

​ linux，把用到了命令过了一遍，shell编程复习了一点。

​ 写了博客，把所有的项目介绍了一遍，把项目都重新回顾了一遍。

​ 传项目代码到github。

4，十天等待

​ 10天每一天都在等待发起面试，但一直没有等到，这段时间就一直在准备前面提到的那些基础和项目。

5， 初试

​ 3月19日：第一次初试。

​ 说他们是腾讯云部门，让我介绍自己最熟悉的项目，我就介绍了博客搭建的项目，主要觉得因为对面是腾讯云，所以就介绍了很多linux的内容。之后面试官问了很多linux命令和一些项目细节，之后问了访问一个网页用到的所有协议和过程，之后问了一些细节，比如握手为啥三次。之后问了进程和线程，之后问了几个数据库查询语句，之后问了红黑树和b+树。还有很多基础的知识，有点记不清了。

​ 3月20日：电话沟通，

​ 问我要了Github账号还有博客主页，之后简单聊了下内容，说给我加一次笔试。

​ 3月24日：笔试（1个小时15分钟）

​ 10道多选题，还有2道编程题。多选题主要是基础知识，但是至少有一半都是课本之外的，有一些运维方面的，比如提高服务器性能的方法。之后也有http状态码，c++语句，五大专业课都有涉及。

​ 编程题，题目一个简单一个难，但主要还是逻辑上的一些过程，没啥深奥的算法。

​ 3月26日：第二次初试

​ 就四个问题：

​ 数据库部署的问题；

​ https加密的过程；

​ 服务器容器的问题；

​ 网页证书的问题；

6，复试

​ 3月29日：第一次复试

​ 自我介绍

​ 问项目，项目的细节，项目的拓展。

​ 之后问了一点基础，都是前面准备的子集。

​ 之后问了数据库的b+树实现，还有STL的一些底层原理。

​ 4月1日：第二次复试

​ 自我介绍

​ 问项目，项目的细节。

​ 之后让你部署淘宝网址，你怎么去部署。

​ 还有一些基础的问题。

7，HR面试

​ 4月2号：主要聊了大学是怎么学习的。

​ 自己性格的优点和缺点。

​ 为啥选择腾讯。

​ 啥时候可以去实习。

​ …………

8，offer call

​ 4月3日：个人信息的确认。

​ 实习期间的待遇。

​ 收到腾讯公司录用函。

9，后记

​ 首先面试要准备充分，要对自己学过的专业知识进行系统的学习，之后还有对他们进行一些拓展，之后要对自己的编程语言很熟悉，多写点算法。之后要找到自己比别人擅长的地方，这一点也很重要。对自己简历上写的项目要特别熟悉，还要做相关的拓展，不清楚的一定不要往上面写。之后面试过程要坦诚，面试官都很nice，你不会的知识点他们都会给你解释，不会的问题一定要谈自己的思路，有时候这种思路更重要。过程中不要紧张，无论初试和复试，面试官都很nice，我感觉就像正常聊天那样。最后就是出了问题从自身找原因，完善自己，努力变得更好。

RIP协议的java实现项目介绍

本项目之前参考过一篇github的部分内容，但现在已经找不到该github，无法注明

1，这个项目实现的功能

​ 模拟了rip协议。

​ 实现了心跳包和数据包的周期性传输。

​ 使用UDP协议进行路由器之间的数据信息传输。

​ 及时更新每个路由器所对应的路由表。

​ 周期性的打印路由表的内容。

​ 删除路由表后其他路由器中本路由器将消失。

​ 使用了NIO编程里面的选择器和通道内容。

2，项目的目录结构

​

3，项目的详细介绍

​ 1，configuration

​ 其中config.txt文件表示一个总的配置文件，包括心跳包发送时间，信息包发送时间，打印路由表的时间

​ 其他的config-x，每一个文件代表一个路由器的直连信息，包括路由器的ID，路由器的入端口，以及路由器的可达路由器以及花费的跳数。

​ 2，parser.java

​ 主要是为了读取配置文件中的信息，将文件中的信息安装固定格式读取后放入相应的数据结构，之后系统使用这些信息进行运作，还实现了一个打印配置文件的函数，主要是为了测试。

​ 3，routingTableEntry.java

​ 这个类实现了一条路由表记录，分别记录了目的路由器，第一跳路由器，路径总花费以及路径经过的所有节点。实现了一个路径的更新函数，保持路径处于最新状态。

​ 4，main.java

​ 首先调用parser类，对配置文件进行读取，接下来new出了一个daemon对象，相当于一个路由器。之后调用daemon的一些方法，启动一些定时器，之后周期性的发送心跳包和信息包来更新路由信息。

​ 5，daemon.java 这是最重要的类，这个系统的核心代码。这里面用了很多NIO编程的知识。

​ 变量上面：

​ 路由器id，出端口，map储存所有的路由表记录，选择器确定当前路由器是否在工作，bool类型的变量为了确定是否发送心跳包和数据包，心跳包计时器，最大跳数，和一些其他变量。

​ 函数上面：

​ 函数一：构造函数，主要初始化路由器的相关变量，更重要的是为每一个入端口DatagramChannel channel，并且设置成非阻塞的，并且打开该通道。最后打开一个新的计时器。

​ 函数二：readConfigFile（）。读配置文件，将心跳包发送间隔，数据包发送间隔。打印间隔。

​ 函数三：isSelected()。如果被选择器选择，那么那么我们可以发送心跳包和数据包。

​ 函数四：sendHeartPackets()。发送心跳包，通过UDP协议。

​ 函数五：sendPackets(）。发送数据包，通过UDP协议。

​ 函数六：createBuffer()。创建buffer，主要是创建数据包的头部和内容，是路由信息，包含目的地，下一跳，路径信息，花费。

​ 函数七：createHeartBuffer()。创建心跳包，包含路由器id信息和额外信息，共8位。

​ 函数八：consistentPacket（）。确定是否是有效的数据包。

​ 函数九：readReceivedPackets（）。将包里的数据读出来。

​ 函数十：heartPackets（）。判断是不是有效的心跳包。

​ 函数十一：updateRoutingTable（）。更新路由表。

​ 函数十二：updateLine（）。更新路由条目，比较cost或者查看是否存在这个路径。

​ 函数十三：startPeriodicTimer（）。startHeartTimer（）。周期的执行任务。

​ 函数十四：displayRoutingTable() 。打印路由表。

​ 函数十五：TimeOut（）。三次心跳包未收到，表示超时。

WIFI人体身份识别

一，项目背景介绍

​ 在大二时，和几个学长一起完成的大学生创新创业项目，当时自己刚学习软件工程一年时间，没有丰富的软件知识，但是由于在老师实验室帮着做过几次类似原理的实验，所以硬着头皮边学习边参与到了这个项目中。这个项目主要应用的时CSI信道容易发生变化的原理，通过路由器发射信号，经过人体反射后对信号产生变化。之后我们采集到了反射后信号（使用了CSI tool软件），经过一系列的预处理后将数据批量的投入一些机器学习的模型中，最后得到了训练结果，当实验者再次进入wifi区域，利用训练结果识别身份。

​ 之前参与的实验和这个项目在原理上是类似的，之前老师提出一个系统，想借助视频，声音以及WIFI信号识别飞入禁飞区的无人机，但是我帮助老师和学长采集实验数据，也是利用CSI信道来识别无人机。

二，实验的设计

​ 1，我们使用路由器在过道一端发射WIFI信号，之后再过道另一端放置经过我们改造的笔记本，主要是将接收天线和笔记本的网卡连接，这样我们在电脑上收到的channel state information（CSI）便直接在电脑上进行析。

我们每次使用三根天线对信号进行发送和接受，每个天线携带30个负载波，所以我们每进行一次实验，可以收到90条CSI信号。

​ 2，每个人在设备间来回走一次，记录为一次实验数据的采集。

三，我主要参与的部分

​ 1，在实验设计阶段，我们设计了实验数据采集的场景，以及设计了需要采集的数据量，以及一些实验数据的保存命名等方面。

​ 2，参与实验数据采集平台的搭建，实验数据的采集主要使用了清华的大学搭建的CISTOOLS平台。我们的实验平台运行在linux系统上面，而我那时候还是个linux小白，停留在知道这个系统的阶段，所以我自学了linux系统的基础命令，了解了linux系统的一些相关知识，参与搭建了实验数据的采集平台。实验平台能自动采集数据，并且借助matlab的图像绘制能力将信号波动显示出来。该实验实验的设备来自其他学长进行类似实验所改装的电脑（将电脑网卡接上天线，更好的收集信息）。

​ 在这个预处理阶段，我们将以上找到的间隔时间段在我们选择的副载波子集上做一个并集，确定了这一次实验我们选定的间隔时间段。

​ 之后根据这个时间段，将这个时间段内的数据置为0，这样形成一个比较稀疏的矩阵，便于后来的处理。之后将选择的副载波子集做一个分段处理，形成一次实验的数据集。

至此，预处理完成

​ 后续设计了Resnet网络，将数据集放入网络进行机器学习，得到模型后进行人体身份识别。

（其实这部分才是这个项目的核心，怪我太菜）

​ *深度学习部分，主要采用了CNN中的ResNet网络，通过卷积和池化获得了特征数据。在CNN进行特征提取之后，我们可以将这些特征理解成单词，而时序上的特征序列就是句子，这样我们可以继续利用RNN做最后的分类工作。

魔塔游戏项目介绍

项目背景：

​ 在学习完c++的课程后，为了更深入的理解c++面向对象的特性，我用c++重写了魔塔游戏，在黑框的形式下进行魔塔游戏，将人物画面设计成了9个符号，并且将地图中的物品（钥匙，血瓶。宝石等）设计成数字存在一个二维数组中，将地图设计了9层，当英雄与不同的物品相遇时，英雄的相关属性会进行改变，并且还使用了一些数学方法，限制了英雄的活动范围，再后来学习完QT框架后，和朋友一起在原有的游戏思路基础上，将游戏改变成了有画面的形式，增加了游戏的可玩性。

ps：黑框式游戏的代码由于时间关系已经不见了只能展示QT版本的游戏

游戏的相关介绍：

黑框阶段：

​ 每个英雄，道具，地图，怪物在表现形式上都是使用9个简单符号构成，为了视觉上的区别，我将这些形象设计了很大的区别。在后台的控制上，用从0开始的数字表示这些英雄，道具，地图，和怪物，以三维数组的方式储存数据，每一个二维数组代表一层地图。在后台根据这些不同数字对英雄的属性进行相对应的调整。最后对怪物的属性进行一些修改，如攻击里和防御力等增加了游戏的可玩性。

QT阶段：

​ 继承了qt的MyLabel类，在该类中设置了所对应英雄和怪物的图片，并且使用setPosition函数统一了所有图片的大小。每一层地图由13*13的小图片构成，使用上下左右方向键控制英雄的移动，在后端仍采用黑框阶段的模式，使用不同的数字来表示不同的内容。右侧显示了英雄的HP、攻击力，防御力和所拥有的钥匙，英雄在移动过程中还能通过使用蓝宝石和红宝石来改变相关属性，最后通过尝试一些方法，最后达到最后一层，打败boss；

游戏的详细分析(QT版本)

游戏文件主要由

magictower.h ,mainwindow.h ,maps.h ,mylabel.h四个头文件，

​ mylable.cpp主要实现了一些放置lable的函数。

将个人博客hexo托管到GitHub

发表于 2019-03-06 | 分类于 博客搭建

将个人博客（hexo前端)托管到GitHub

​ 用了大概有一天的时间，将这个新博客搭建完了，总的来说，将博客托管到Ggithub上面相对于之前的旧博客搭建更简单一点

一，搭建本博客需要的一些前提知识和内容

  1. 需要你自己拥有一个github账号，并且了解一点github的基本知识。
  2. 需要对HEXO的博客前端有一定的了解，可以参考官网。
  3. 需要熟悉linux端的一些基本命令。
  4. 需要拥有自己的一个域名。

二，本人linux环境的一些说明

  • 采用虚拟机的方式，在虚拟机中安装了centos系统。
  • 启动虚拟机，并连接网络，使用ifconfig命令查看系统的IP地址。(局域网的ip)
  • 使用xshell软件登陆虚拟机里的centos系统的root用户。
  • 个人比较喜欢linux这种纯手打操作的操作系统，用起来比较方便。

三，安装的一些基本步骤

  • yum install git
  • yum install nodejs（hexo基于node.js)
  • yum install npm(npm是js的包管理工具)
  • npm install -g hexo-cli（安装hexo的包）
  • hexo init myblog 其中myblog是一个名称，也是你博客所在的文件夹
  • cd myblog
  • npm install

此时博客已经安装在了你的服务器上，接下来在github上创建该博客的代码仓库

  • 在github上建立名为 yourgithub.github.io的代码仓库 yourgithub是你的GitHub名称

  • 回到bash中

  • ssh-keygen -t rsa -C "youremail"

  • 此时会生成一个.ssh文件夹，根据提示cd到该文件夹中

  • cat id_rsa.pub 将里面的文字全部复制到剪切板，打开github

  • 进入github->setting->ssh keys-> new ssh key ,将剪切板的文件粘贴进去，保存

  • 进入bash

  • vim _config.yml

  • 在vim中 查找（/+查找的字段）deploy字段，将里面的YourgithubName改为你的github名。

  • hexo clean

  • hexo g

  • hexx d

  • 以上三个命令在每次修改文件或者写作后都应该使用，作用是更新网页

  • 执行第三条命令时需要你输入你的github名称以及你的密码

此时已经能在任一浏览器中浏览网页，但还需要绑定域名

  • 绑定域名在各平台是傻瓜操作，主要有以下需要注意的：
    • 记录类型为CNAME
    • 记录值为你的代码仓库名称，即yourgithub.github.io
  • 进入github在该仓库下点击setting，设置custom domain为你的域名。
  • 进入bush，进入source目录
  • vim CNAME,在vim中写上你的域名。
  • 执行之前提过的三条命令。

四、关于写作和其他配置

到现在，整个博客已经安装完毕了，错略的总结了一下基本步骤，关于如何进行博客写作还有对博客主题修改以及其他的一些更高级的配置，网络上有很多好的博文，就不再赘述了，主要还是因为现在要找工作时间有限，以后如果有空余时间了，还是会过来再写一些的

推荐大家使用next主题的第四个样式，好评！！！！

如果你使用了next主题，想要更好的装饰博客，推荐看博文

五、结束

感谢github的这个功能，如果你想要基于服务器自己搭建一个博客，请看我的另一篇博文，但是那种方式成本比较高，你还需要自己维护自己的服务器。
